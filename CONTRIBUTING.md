# Вклад в Log Owl

Этот документ кратко фиксирует правила флоу, веток и коммитов.

---

## Базовый флоу

- Разработка ведётся **от ветки `dev`**.
- Ветка `main` предназначена для стабильных релизов и релизных candidate‑сборок.

---

## Имена веток

Для имён веток используется один из вариантов:

- `main`
- `dev`
- `<type>/<kebab-case-name>` (например, `feature/add-time-entries-filter`),

где `<type>` — один из `feature`, `fix`, `chore`, `docs`, `refactor`, `test`, `ci`.  
Соответствие имени ветки этому шаблону проверяется в pre‑commit скриптом `scripts/check-branch-name.js`.

---

## Сообщения коммитов

Для сообщений коммитов используется соглашение **Conventional Commits** (проверяется `commitlint` через хук `commit-msg`).
